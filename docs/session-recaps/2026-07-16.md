## 3. Décisions & raisonnement clés

- **Socle d'abord, puis features** : le socle (i18n/config/embeds) rend caduques les propositions « personnalisation » des 48 features ; les features consomment le socle.
- **Worktrees isolés sur `origin/main`** pour tout build : le working tree des bots contient du WIP utilisateur (`.ai/`, docs, `auto-rename*.json`) que je ne dois jamais committer, et les branches feature avaient divergé (doublon SEC-001 pré-PR #50). Chaque build se fait dans un worktree propre → commit → push `main` en fast-forward.
- **Migrations Neon** : le migrator drizzle applique par **timestamp** (`when > max(created_at)`), pas par hash. J'ai enregistré chaque nouvelle migration avec son `when` exact. Découverte : les baselines `__drizzle_migrations` sont **incohérentes en fin de ligne** (Coverioos/Collabioos = hash CRLF, ReNamioos = LF, Moodioos = mixte) alors que le repo stocke du LF → dette d'infra à assainir (issue créée, §7).
- **Board Collabioos #59 vs #36** : décision produit = **canoniser sous la catégorie « Collab »** (comportement actuel conservé, aucun changement).
- **Anti-dup / DDD** respectés : ports/adapters, domaine pur (purity test exécutable), embeds centralisés via la fabrique, i18n catalogué.
